# 去 AI 味与发布前检查：让稿子像你本人说的

## 核心原则

AI 初稿的问题通常不是结构不完整，是太整齐、太概括、太像正确答案。小红书要的是个人经历、具体场景、口语节奏、可感知细节。

改稿分两步：先人格化，再平台规则检查。人格化 = 加"我怎么知道、我踩过什么、我具体怎么做"；规则检查 = 减绝对化表达、收益承诺、过度导私聊等风险表达。

发布前质量检查：开头给用户入场理由，正文少废话有结构易吸收，结尾引导评论或收藏。真人感不等于流水账，必须同时服务点击率、停留时长、互动率。

目标不是把文案变花，是让它更像账号本人能说出口的话，同时更容易被点开、看完、收藏、评论。

所谓 AI 味，常来自"太顺、太满、太正确"——用个人语言、真实意图、具体证据把它拉回人。一篇笔记只能有一个核心机制；开头 5 秒必须独立成立，含话题、继续看的理由、可信证据。

改稿时还要判断这篇适合单图 / 长文 / 6-8 页图文；要做图文就得拆成封面、痛点、认知、方法、案例、操作、总结、引导等页面任务，每页只讲一个主要信息。

## 什么时候用

1. 有 AI 初稿，觉得不像自己。
2. 发布前做最后检查。
3. 想把太书面太整齐的文案改成口播感。
4. 担心表达过度承诺或过度导流。
5. 觉得开头不抓人、正文太散、没收藏点或评论点。

## 执行步骤

1. **标 AI 味来源**：查抽象词、整齐排比、泛泛建议、缺个人证据、书面连接词。列 3-10 个具体问题点。

2. **内容四维诊断**：查文字干净度、封面标题承诺、表达效率、认知差或新鲜感。先判问题在"看不进去 / 看不懂 / 没新意 / 没证据"哪类。

3. **单一核心机制审计**：列正文所有观点 / 方法 / 故事 / 结论，只留一个主机制，其余当证据或删。用户看完能复述"这篇到底讲一件什么事"。

4. **查 5 秒开头**：第一屏同时答——讲什么、为什么现在看、凭什么信你。开头离开正文也能成立，不靠后文补解释。

5. **查爆款结构**：有无钩子、真实背景、3 点内核心信息、总结、评论 / 收藏引导。能指出卡在点击 / 停留 / 互动哪。

6. **加真人材料**：要求用户提供或据已有信息提取真实经历、具体场景、踩坑、数据、对话、前后对比。每关键观点至少一具体细节支撑。

7. **提清晰度与易吸收度**：删铺垫废话，核心压到 1 方法 / 1 坑 / 1 对比或 3 点内。用户前 3 秒 / 首行就知道为什么继续看。

8. **改成账号口吻**：按账号档案调句长、语气、词汇、节奏。用户能判"这像我会说的话"。

9. **做发布检查**：查绝对化表达、收益承诺、过度导外部联系、夸张标题、不必要敏感表达。输出"可发 / 需修改 / 暂不建议发"判断和修改点。

10. **转图文页面结构**：要发图文按 3:4 手机端拆 6-8 页——封面、痛点、认知、方法、案例、操作、总结、引导。每页有页面任务、标题、副标题、核心文案、视觉重点、互动目标，正文不硬塞单页。

11. **视觉可读性检查**：标题够大否、文字过多否、封面是否只一钩子、内页有收藏价值否、是否像普通 PPT。输出需视觉导演处理的字段，不只给纯文本。

## 边界

- 用户还没正文只想要选题 → 不用。
- 用户要改核心观点或重做定位 → 转对应 skill。

## 失败模式

- 只把句子改口语，不补真实材料。
- 为更像真人牺牲信息密度。
- 只靠自动检查，不做人工过目。
- 一味堆干货，用户没情绪 / 收藏 / 评论理由。
- 开头绕太久，用户没看到价值就划走。
- 一篇塞多个主机制，用户不知该记什么。
- 只润色不先说问题来自标题承诺 / 表达效率 / 证据 / 结构。
- 把长文硬塞图文，每页字太多手机端不可读。
- 只改正文不判封面 / 内页 / 结尾页各自任务。
