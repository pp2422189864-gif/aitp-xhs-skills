# 排期与周复盘：把灵感驱动变成数据驱动

## 核心原则

小红书冷启动不是单篇内容问题，是连续发布和反馈问题。排期解决"今天发什么、几点发、发哪类"；复盘解决"数据说明什么、下周怎么改"。

新号要分阶段：先让账号行为自然，再让平台识别标签，再逐渐稳定内容和商业转化。每周记录曝光 / 点击 / 赞 / 藏 / 评 / 转 / 涨粉，并让 Agent 把结论写回记忆。

冷启动实验节奏：前 10 条是系统画像建立期，建议组合——自我背景 / 经历、方法 / 经验、踩坑 / 反思、情绪 / 观点。定位别只靠想，用 10-20 条内容验证。复盘别只问"爆没爆"，要问：系统认识你了没、哪类易被推荐、哪类易被互动和关注。

排期 + 复盘合起来，内容系统才从灵感驱动变成数据驱动。而且每周不只产排期，要把"验证过的内容类型、禁用表达、评论区需求、下周实验假设"写回账号档案——下一轮不是重来，是在上一轮证据上迭代。

图文生产也要排进节奏：哪条做单图、哪条做 6-8 页、哪天出视觉确认图、哪天批量出终图、哪周复盘封面点击和内页收藏。

## 什么时候用

1. 有选题库但不知道怎么排一周 / 一月。
2. 发了几篇不知道数据怎么解释。
3. 想从冷启动进入稳定更新。
4. 想让 Agent 记住每周复盘规律。
5. 刚开号，想设计前 10 条。
6. 发了 10-20 条想据数据收敛定位。

## 执行步骤

1. **确定阶段**：判断账号处于画像建立 / 方向测试 / 标签识别 / 内容放大 / 稳定转化哪一阶段。每阶段发布目标不同。

2. **设计前 10 条画像内容**：新号优先 3 条自我背景 / 经历、3 条方法 / 经验、2 条踩坑 / 反思、2 条情绪 / 观点。系统和用户都看到一个"完整的人"，不是单一教程机器。

3. **生成排期表**：按周安排内容类型、发布时间、标题方向、制作任务。至少给 7 天，可扩到 30 天。

4. **安排图文制作任务**：每条标注形态（单图 / 6-8 页 / 长文 / 案例拆解 / 工具清单）；要图时安排 10 问澄清、视觉母版、确认图、终图、审查时间。排期含内容生产和视觉生产，不只发布日期。

5. **建数据看板**：记录曝光、点击率、停留、赞、藏、评、转、关注、私信、发布时间、内容类型。用户能每周填。

6. **补视觉数据字段**：记录封面风格、标题字数、主视觉类型、页数、是否有统一母版、收藏率、评论主题、用户截图反馈。能判断问题出在选题 / 封面 / 内页密度 / 定位。

7. **10-20 条后收敛定位**：比较不同主题在点击 / 停留 / 互动 / 关注上的差异，定主方向和备选。用一轮实验数据收敛，不凭感觉。

8. **复盘并回写记忆**：判断最佳类型、最佳时间、失败原因、评论区新选题、爆了是否顺着发。输出下周 3-5 个具体调整动作，建议写入账号档案。

9. **生成周度运营报告**：汇总本周有效假设 / 失效假设 / 最强标题触发器 / 最强内容机制 / 最值得继续测的 3 方向。报告能直接更新档案、选题库、下周排期。

## 边界

- 只有一个初稿要改 → 用去 AI 味 skill。
- 没有任何选题或定位 → 先补前置 skill。

## 失败模式

- 排期过满，新手执行不了。
- 只看赞，不看藏 / 评 / 点击 / 涨粉关系。
- 复盘只写感受，没转成下周动作。
- 发 3 篇没起色就放弃，样本太少无法判方向。
- 一篇爆了就乱发，打断系统归类。
- 每周复盘只留聊天记录，没写回档案或下周实验。
- 排期只写发布时间，不预留封面 / 拆页 / 确认 / 审查时间。
- 复盘只看选题，不记封面风格 / 页数 / 密度 / 收藏反馈。
