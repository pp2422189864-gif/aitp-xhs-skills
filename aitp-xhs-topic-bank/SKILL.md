# 七类选题库：别每天从空白开始想

## 核心原则

选题焦虑来自每天从零开始。解决办法是把选题变成类型化生产：痛点、数字、对比、稀缺、共鸣、资源、反常识。

每类作用不同：痛点型命中问题；数字型制造清晰感；对比型凸显变化；稀缺型制造注意力；共鸣型建立信任；资源型提高收藏；反常识型制造转发理由。

Agent 不该自由发挥，应在账号档案和对标公式约束下按类型生成可筛选选题。

每条内容先答三问：解决谁的问题、用户为什么点进来、看完能带走什么。标题公式只是入口，真正决定放大的是这条内容满足了**情感共鸣 / 见证成长 / 获得新知 / 学习提升 / 身份表达**中的哪类底层需求。

标题要"可追溯"：每个标题能说清来自哪类触发器、为什么适合这话题、有没有留悬念。不是越炸越好，而是在 20 个中文字内同时命中真实痛点、扩大受众词、至少含 2 个张力元素。

选题库不只输出文字标题，还要判断每个选题适合哪种图文形态、封面钩子、主视觉、收藏理由。好选题要能落成 3:4 竖版封面 + 6-8 页滑动结构，不只停在一句标题。

## 什么时候用

1. 已有定位，但每天不知道发什么。
2. 要为一个赛道批量生成 20-50 个选题。
3. 觉得 AI 标题太像营销号，需要用公式约束。
4. 想建可持续内容栏目。
5. 想知道某题为什么值得点 / 看 / 藏 / 转。

## 执行步骤

1. **读账号约束**：确认目标用户、赛道关键词、变现路径、人设语气、禁区。缺这些先调档案或定位 skill。

2. **建七类公式**：为痛点 / 数字 / 对比 / 稀缺 / 共鸣 / 资源 / 反常识各生成 3-5 个模板。每模板含可替换变量。

3. **标标题触发器**：给每候选标题标主触发器（认知冲突 / 好奇缺口 / 损失提醒 / 身份代入 / 数字锚定 / 结果承诺 / 案例证明 / 争议互动 / 场景条件 / 行动号召 / 权威借力 / 互动测试）。每标题有"类型来源 + 推荐理由"，不自由发挥。

4. **标题五项硬检**：≤20 中文字、留悬念、不说完答案、用更大用户词、击中真实痛点、至少 2 个张力元素。过长 / 过窄 / 太平 / 太像结论必须重写。

5. **叠五类用户需求**：为每选题标主需求（情感共鸣 / 见证成长 / 获得新知 / 学习提升 / 身份表达）。每选题能说明用户为什么点 / 看 / 藏 / 评 / 转。

6. **三问校验**：逐条答——解决谁的问题、为什么点进来、看完带走什么。答不清降级或重写。

7. **批量生成**：每类 5 个，附标题、目标读者、内容形态、收藏理由。至少 35 候选。

8. **筛发布优先级**：按变现相关度 / 用户痛感 / 制作难度 / 收藏价值 / 身份表达强度打分。选出本周 3-5 优先题。

9. **补视觉交付字段**：为优先题补内容类型、传播目标、读者情绪、信息密度、封面钩子、主视觉方向、推荐风格、页数建议。每个优先题能继续交给视觉导演生成 3:4 封面或 6-8 页图文。

## 边界

- 用户没定位或目标人群 → 不用。
- 用户要完整正文而非选题标题 → 不用。

## 失败模式

- 让 AI 自由发挥标题，表达过度夸张。
- 只生成标题，不说明内容形态和读者收益。
- 选题全是热点，和变现路径无关。
- 每条追信息密度，却没情绪 / 身份 / 成长证据。
- 标题好点，正文没带走感。
- 标题无法追溯任何触发器，只"像爆款"。
- 标题说完答案、话题太窄或超 20 字。
- 选题只有文字标题，没封面钩子 / 主视觉 / 滑动页结构。
