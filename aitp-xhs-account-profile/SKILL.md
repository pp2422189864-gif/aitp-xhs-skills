# 账号档案：让 AI 真正认识你这个号

## 核心原则

Agent 的价值不是回答一次问题，而是长期协作。长期协作的前提是它知道这个号是谁、说什么、不说什么、为什么可信、已经验证过什么。

账号档案把定位、变现模式、人设语气、内容边界、已有数据、对标公式写成一份可读取文件。之后每次生成选题 / 标题 / 正文 / 复盘，Agent 不用重新猜你是谁。

做个人 IP 的人要明白：用户关注的不仅是观点，而是"这个人是谁、怎么用工具、为什么信他、我是不是也想成为他"。所以档案不能只写赛道，还要写真实经历、信任资产、成长线、生活方式边界、人设延展规则。

一份好档案 = 风格指南 + 边界文件 + 信任名片 + 运营日志。

补充两个关键字段：
- **个人语言样本**：你真实会说的 5-10 句话、常用句式、禁用词、不想像谁的账号。后续改稿能判断"像不像你本人"。
- **可信主张清单**：你能公开证明的观点、结果、案例、offer，以及不能承诺什么。内容不只靠语气取信，要有证据和边界。
- **视觉身份档案**：账号长什么样也要记——默认画幅、主风格、配色令牌、字体系统、封面规则、内页组件、禁用风格。图文时代，视觉也是人设的一部分。

## 什么时候用

1. 每次让 AI 写内容都觉得不像自己。
2. 想把定位 / 语气保存成长期上下文。
3. 正在搭一人公司或个人 IP 内容系统。
4. 已有数据 / 踩坑记录，想让 Agent 下次自动参考。
5. 想从"内容垂直"过渡到"人设垂直"但怕发散。
6. 想把经历 / 专业 / 审美 / 生活整理成可复用表达资产。

## 执行步骤

1. **收集基础字段**：定位、目标用户、变现模式、人设关键词、内容边界、已有数据。每字段至少一句明确描述。

2. **补信任资产**：提取真实经历、专业背景、作品 / 结果、成长线、价值观、生活方式、可公开边界。能回答"为什么用户信这个人而不是只收藏这条内容"。

3. **建个人语言样本**：收 5-10 句你真实发过的话、常用表达、禁用词、句长偏好、情绪边界、不想像谁。另一个 Agent 能判断一句话像不像本人。

4. **建可信主张清单**：写清可公开主张什么、有什么证据、哪些结果不能承诺、哪些 offer 可被内容自然承接。后续内容有证据有边界。

5. **定人设垂直规则**：区分早期内容垂直和中期人设垂直——哪些话题是主线、哪些可被主线接住、哪些放小号或不发。垂直 = 围绕同一个人设语境延展，不是只能发单一题材。

6. **建视觉身份规则**：记录默认画幅、主视觉风格、辅助风格、色彩令牌、字体系统、封面标题比例、内页组件、图标线条语言、禁用视觉反模式。另一个 Agent 能判断封面像不像你、哪些风格不该用。

7. **写成账号档案**：输出 `账号档案.md`，含定位卡、信任资产、个人语言样本、可信主张、语气规则、视觉身份、禁区、内容栏目、对标公式、数据日志、人设延展规则。另一个 Agent 只读此文件也能判断是否该写某类内容。

8. **生成记忆写入指令**：给可直接对 WorkBuddy 说的指令，如"请记住以下账号信息，并在后续内容生成中自动参考"。用户可复制使用。

9. **建更新机制**：每周把表现最好的内容、失败原因、禁用表达、评论区认可点、私信问题、好封面、失效视觉风格加入档案。档案是可迭代资产，不是静态简介。

## 边界

- 只需临时写一条与账号无关的文案 → 不用。
- 还没有基本定位 → 先用变现倒推 skill。

## 失败模式

- 档案只写口号不写边界。
- 只写"温柔专业真诚"等形容词，没可判断示例。
- 建完不更新数据和经验。
- 只写内容标签，不写这人为什么可信 / 可亲近 / 可持续关注。
- 人设延展过宽，系统用户都找不到主线。
- 没有个人语言样本，改稿只能凭抽象形容词猜口吻。
- 没有可信主张和 offer 边界，内容越写越空。
- 只记文案语气，不记封面配色字体页面结构。
- 每条图文都换视觉模板，用户记不住账号。
